# 사고 시각화 루틴 웹 플랫폼 기획 초안

## 1. 목표

교사가 학급, 학생, 과목, 모둠, 활동을 관리하고, 학생은 활동 코드/QR 또는 학생 접속 정보로 활동에 참여한다. 활동은 Project Zero의 사고 시각화 루틴을 웹 인터페이스로 구현한 형태이며, 교사는 학생 결과물을 조회하고 AI 분석 보고서를 활용해 수업 피드백과 후속 활동을 설계한다.

## 2. 현재 가정

- 초기 플랫폼은 개인 교사형으로 만든다.
- 학교 관리자, 기관 관리자, 교사 간 공유 기능은 초기 범위에 포함하지 않는다.
- 학생은 일반 회원가입을 하지 않는다.
- 학생은 교사가 등록한 명단을 기반으로 인증한다.
- 첫 구현 루틴은 See-Think-Wonder 개인 활동형이다.
- 모둠 기능은 초기부터 핵심 기능으로 포함한다.
- AI 분석은 교사가 먼저 검토하고, 필요한 경우 학생에게 공개한다.

## 3. 성공 기준

초기 버전은 다음 흐름이 동작하면 성공으로 본다.

1. 교사가 로그인한다.
2. 교사가 엑셀 또는 CSV로 학생을 일괄 등록한다.
3. 시스템이 학급을 자동 생성하고 학생별 비밀번호를 자동 발급한다.
4. 교사가 과목 또는 학급 기준으로 활동을 만든다.
5. 교사가 See-Think-Wonder 활동을 생성한다.
6. 학생이 QR/활동 코드 또는 내 활동 목록을 통해 활동에 접속한다.
7. 학생이 See / Think / Wonder 카드를 작성하고 제출한다.
8. 교사가 학급 전체 결과와 학생별 결과를 조회한다.
9. 교사가 카드 태그와 AI 분석 보고서를 활용한다.

## 4. 사용자 유형

### 4.1 교사

- 개인 계정으로 가입/로그인한다.
- 자신의 학급, 학생, 과목, 모둠, 활동을 직접 관리한다.
- 학생 결과물을 조회하고 AI 분석을 실행한다.

### 4.2 학생

- 일반 회원가입을 하지 않는다.
- 교사가 등록한 정보로 접속한다.
- 접속 정보는 학번, 이름, 비밀번호이다.
- 활동 목록, 개인 활동, 자신이 속한 모둠 활동을 조회할 수 있다.

## 5. 학생 등록과 인증

### 5.1 학생 일괄 등록

- 교사는 엑셀 또는 CSV 파일로 학생을 일괄 등록한다.
- 업로드 파일에는 학급, 학번, 이름이 포함된다.
- 파일에 포함된 학급이 없으면 시스템이 자동 생성한다.
- 같은 학급명이 이미 있으면 기존 학급에 학생을 추가한다.
- 학번은 전체 고유값으로 사용한다.

예상 업로드 양식:

```text
학급 | 학번 | 이름
1학년 3반 | 20260123 | 김민준
1학년 3반 | 20260124 | 이서연
1학년 4반 | 20260125 | 박지후
```

### 5.2 학생 비밀번호

- 학생 비밀번호는 시스템이 자동 생성한다.
- 학생은 비밀번호를 직접 변경할 수 없다.
- 교사만 학생 비밀번호를 재발급할 수 있다.
- 교사는 학생별 비밀번호를 확인하거나 다운로드할 수 있어야 한다.

### 5.3 학생 인증

학생은 다음 정보로 인증한다.

```text
학번 + 이름 + 비밀번호
```

## 6. 교사 운영 방식

교사는 가입 또는 초기 설정에서 운영 방식을 선택한다. 운영 방식은 설정에서 변경할 수 있다.

### 6.1 학급 우선

- 담임교사에게 적합하다.
- 학급 안에서 과목과 활동을 관리한다.

예:

```text
5학년 2반
└─ 국어
   └─ 활동
```

### 6.2 과목 우선

- 중등 교사 또는 초등 전담교사에게 적합하다.
- 과목을 만들고 활동마다 대상 학급을 선택한다.
- 하나의 활동을 여러 학급에 배정할 수 있다.
- 활동 자체는 하나로 유지하고, 결과 화면에서 학급별 필터로 확인한다.

예:

```text
과학
└─ See-Think-Wonder 활동
   ├─ 5학년 1반 결과
   └─ 5학년 2반 결과
```

## 7. 활동 자료

초기 지원 자료 형식은 다음과 같다.

- 텍스트
- 이미지
- PDF

링크는 초기 범위에서 제외한다.

자료 개수와 배치는 전체 공통 규칙으로 고정하지 않고, 각 사고 루틴을 구현할 때 루틴 특성에 맞게 결정한다.

## 8. 활동 상태와 접속

### 8.1 활동 상태

활동 상태는 두 가지이다.

- 진행 중
- 마감됨

교사는 버튼으로 활동을 마감할 수 있다. 마감된 활동은 다시 열 수 있다.

학생 권한:

- 진행 중: 작성, 수정, 삭제, 제출 가능
- 마감됨: 읽기 전용

### 8.2 활동 접속

활동마다 다음을 자동 생성한다.

- 짧은 활동 코드
- QR 코드

학생 접속 경로는 두 가지이다.

#### 경로 1: 활동 바로 접속

```text
QR 또는 활동 코드 입력
→ 학번 + 이름 + 비밀번호 인증
→ 해당 활동으로 이동
```

#### 경로 2: 내 활동 목록 접속

```text
학생 접속 페이지
→ 학번 + 이름 + 비밀번호 인증
→ 자신이 속한 학급에 배정된 활동 목록 조회
→ 활동 선택
```

학생의 내 활동 목록에는 진행 중 활동과 마감된 활동을 모두 표시한다.

- 진행 중 활동: 참여/수정 가능
- 마감된 활동: 읽기 전용 결과 확인 가능

## 9. 교사 활동 목록

교사 활동 목록에는 상태 뱃지와 상태 필터를 모두 제공한다.

상태 뱃지:

- 진행 중
- 마감됨

상태 필터:

- 전체
- 진행 중
- 마감됨

## 10. 모둠 기능

모둠 기능은 초기부터 핵심 기능으로 포함한다.

### 10.1 모둠 생성

교사는 학급별로 모둠을 만들 수 있다.

지원 방식:

- 직접 모둠 생성
- 학생 직접 배정/이동
- 자동 모둠 생성
- 자동 생성 후 수동 수정

자동 생성 기준:

- 무작위 배정
- 모둠당 인원수 기준
- 모둠 수 기준

### 10.2 기본 모둠과 활동별 모둠

모둠은 다음 두 수준으로 관리한다.

- 학급 기본 모둠
- 활동별 모둠

활동별 모둠을 사용할 때는 학급 기본 모둠을 활동에 복사한 뒤, 활동 안에서만 수정한다. 활동별 수정은 학급 기본 모둠에 영향을 주지 않는다.

### 10.3 학생 모둠 정보 표시

학생은 다음 위치에서 자신의 모둠 정보를 확인할 수 있다.

- 내 활동 목록
- 활동 화면

개인 활동에서는 모둠 정보를 작게 표시하거나 숨길 수 있다. 공동 활동에서는 모둠과 모둠원 정보를 명확히 표시한다.

### 10.4 공동 작성 방식

공동 작성 방식은 루틴별로 결정한다.

가능한 방식:

- 모둠 공용 보드형
- 개인 작성 후 모둠 종합형
- 역할 분담형

공동 작성 결과물의 작성자 표시도 활동 또는 루틴 설정에 따라 결정한다.

가능한 설정:

- 작성자 표시
- 교사에게만 작성자 표시
- 익명 협업

## 11. 첫 사고 루틴: See-Think-Wonder

### 11.1 기본 형태

첫 구현 루틴은 See-Think-Wonder 개인 활동형이다.

학생은 다음 세 영역에 카드를 여러 개 추가한다.

- See
- Think
- Wonder

각 카드는 짧은 문장 또는 생각 단위이다.

### 11.2 활동 자료

See-Think-Wonder 활동 생성 시 자료는 다음 구조를 사용한다.

- 활동 제목
- 대상 학급
- 안내 텍스트
- 중심 자료 1개
  - 이미지 또는 PDF

### 11.3 학생 작성 흐름

- 학생은 See / Think / Wonder 카드를 여러 개 작성할 수 있다.
- 카드 추가, 수정, 삭제는 자동 저장된다.
- 학생은 제출 버튼을 눌러 완료 상태를 표시한다.
- 제출 후에도 활동 마감 전까지 수정할 수 있다.
- 제출 후 수정하면 교사 화면에 제출 후 수정됨 표시를 한다.
- 마감 후에는 읽기 전용이다.

### 11.4 학생 화면 구조

반응형 혼합 구조를 사용한다.

데스크톱/태블릿:

- 자료 왼쪽
- See / Think / Wonder 입력 오른쪽
- See / Think / Wonder는 세 컬럼으로 표시

모바일:

- 자료 위
- 입력 아래
- See / Think / Wonder는 탭 방식으로 표시

## 12. See-Think-Wonder 교사 결과 화면

### 12.1 학급 전체 보기

- 기본 결과 화면은 학급 전체 보기이다.
- See / Think / Wonder 컬럼별로 모든 학생 카드를 모아 보여준다.
- 기본은 학생 이름을 표시한다.
- 익명 보기 토글을 제공한다.
- 익명 보기에서는 학생 이름과 학번을 숨긴다.
- 카드 또는 학생 이름을 클릭하면 해당 학생 개인 결과 화면으로 이동한다.

### 12.2 학생별 보기

- 한 학생의 See / Think / Wonder 카드를 한 화면에서 조회한다.
- 상단에 이전 학생, 다음 학생 화살표 버튼을 제공한다.
- 교사는 학생별 결과를 빠르게 넘겨보며 확인할 수 있다.

### 12.3 카드 태그

교사는 학생 카드에 태그를 붙일 수 있다.

기본 태그:

- 좋은 관찰
- 흥미로운 질문
- 오개념
- 토론거리

교사는 직접 태그를 추가할 수 있다.

태그 기능:

- 카드에 태그 지정
- 태그별 필터링
- 태그 공개 여부 선택

태그 공개 정책:

- 기본은 교사만 볼 수 있다.
- 교사가 공개하면 학생이 자신의 결과 화면에서 공개된 태그를 확인할 수 있다.

교사 수동 코멘트 기능은 초기 범위에서 제외한다. 피드백은 AI 분석 결과로 제공한다.

## 13. AI 분석

### 13.1 기본 정책

- AI 분석 결과는 기본적으로 교사에게만 표시한다.
- 교사가 승인하면 학생에게 공개할 수 있다.
- 학생에게 공개되는 것은 학생용 피드백이다.
- 교사용 해석은 학생에게 공개하지 않는다.
- 생성된 AI 분석 결과는 저장한다.
- 학생 제출물이 수정되면 갱신 필요 표시를 한다.
- 다시 분석하면 기존 결과를 새 결과로 교체한다.
- 분석 이력 버전 관리는 초기 범위에서 제외한다.

### 13.2 개인 AI 분석

개인 AI 분석은 학생이 제출한 다음에만 실행할 수 있다.

상태:

- 학생 제출 전: 분석 버튼 비활성화
- 학생 제출 후: 분석 버튼 활성화
- 제출 후 학생이 수정함: 분석 갱신 필요 표시

개인 분석 보고서는 학생 친화형 피드백과 교사용 해석으로 분리한다.

학생용 피드백:

- 쉬운 표현
- 잘한 점
- 더 생각해볼 점
- 다음 질문

교사용 해석:

- 관찰/해석/질문 사용 방식
- 사고 특징
- 지도 시 참고할 점

### 13.3 학급 전체 AI 분석

학급 전체 AI 분석은 교사가 언제든 실행할 수 있다. 보고서에는 제출률을 함께 표시한다.

예:

```text
24명 중 18명 제출, 제출률 75%
```

제출률이 낮으면 일부 학생 결과 기준임을 표시한다.

학급 AI 분석 보고서 내용:

- 많이 나온 관찰
- 흥미로운 질문
- 오개념 가능성
- 더 탐구할 주제
- 다음 발문 제안
- 추천 사고 루틴
- 제출률 표시

학급 전체 분석 보고서는 교사 결과 화면의 학생 제출물 테이블 위 또는 아래에 표시한다.

## 14. 아직 결정하지 않은 항목

다음 항목은 이후 루틴 구현 또는 상세 설계 단계에서 결정한다.

- See-Think-Wonder 이후 두 번째 사고 루틴
- 공동 작성 루틴의 첫 구현 대상
- 루틴별 자료 개수와 배치
- 학생 결과물 내보내기 방식
- 교사용 보고서 PDF/CSV 내보내기 여부
- AI 분석에 사용할 구체적 평가 기준과 프롬프트 구조
- 교사 대시보드의 통계 범위
- 활동 추천 기능의 상세 기준
- 학생 비밀번호 안내/출력 화면 방식
- 엑셀/CSV 업로드 오류 처리 규칙
