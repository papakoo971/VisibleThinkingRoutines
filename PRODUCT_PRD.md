# 사고 시각화 루틴 웹 플랫폼 PRD

## 1. 제품 개요

### 1.1 제품명

가칭: Visible Thinking Routines Platform

### 1.2 제품 목표

교사가 Project Zero의 사고 시각화 루틴을 수업 활동으로 만들고, 학생이 웹에서 개인 또는 모둠 단위로 참여하며, 교사가 결과물과 AI 분석 보고서를 바탕으로 수업 피드백과 후속 활동을 설계할 수 있는 통합 플랫폼을 만든다.

### 1.3 핵심 문제

교실에서 사고 루틴은 종이 활동지, 포스트잇, 칠판, 패들렛류 도구 등으로 운영할 수 있지만 다음 문제가 있다.

- 학생별 사고 과정이 누적 관리되기 어렵다.
- 교사가 학급 전체 결과를 빠르게 구조화해서 보기 어렵다.
- 개인 결과와 모둠 결과를 함께 관리하기 어렵다.
- 학생 산출물에 대한 분석과 후속 발문 설계가 교사 개인 부담으로 남는다.
- 사고 루틴별 활동 자료, 제출물, 피드백, 추천 활동이 하나의 흐름으로 연결되지 않는다.

### 1.4 제품 방향

이 제품은 단순한 루틴 목록 또는 활동지 생성기가 아니라, 사고 루틴의 단계 구조를 웹 인터페이스로 바꾸고 학생의 사고 흔적을 데이터화하는 수업 운영 플랫폼이다.

## 2. 사용자와 권한

### 2.1 교사

교사는 개인 계정으로 가입하고 로그인한다.

교사 권한:

- 학급 생성/수정/삭제
- 학생 엑셀/CSV 일괄 등록
- 학생 비밀번호 재발급
- 과목 또는 학급 기준 활동 관리
- 학급 기본 모둠 관리
- 활동별 모둠 관리
- 사고 루틴 활동 생성
- 활동 QR/활동 코드 확인
- 활동 마감/다시 열기
- 학생 제출물 조회
- 카드 태그 관리
- AI 개인 분석 실행/조회/학생 공개
- AI 학급 분석 실행/조회

초기 범위에서 제외:

- 학교 관리자
- 교사 간 공동 관리
- 기관 단위 계정
- 교사 간 활동 템플릿 공유

### 2.2 학생

학생은 일반 회원가입을 하지 않는다. 교사가 등록한 학생 명단을 기반으로 접속한다.

학생 인증 정보:

```text
학번 + 이름 + 비밀번호
```

학생 권한:

- 자신에게 배정된 활동 목록 조회
- 진행 중 활동 참여
- 마감된 활동 결과 읽기
- 자신의 개인 제출물 조회
- 자신이 속한 모둠 정보 조회
- 공개된 태그 조회
- 교사가 공개한 AI 개인 피드백 조회

학생 제한:

- 비밀번호 직접 변경 불가
- 다른 학생 결과물 조회 불가
- 교사용 태그와 교사용 AI 해석 조회 불가

## 4. 핵심 사용자 흐름

### 4.1 교사 초기 설정

```text
교사 가입
→ 로그인
→ 운영 방식 선택
→ 학생 명단 업로드
→ 학급 자동 생성 확인
→ 학생 비밀번호 확인/다운로드
→ 학급 기본 모둠 생성
```

운영 방식:

- 학급 우선
- 과목 우선

학급 우선은 담임교사에게 적합하다. 과목 우선은 중등 교사 또는 초등 전담교사에게 적합하다.

### 4.2 학생 일괄 등록

교사는 엑셀 또는 CSV 파일을 업로드한다.

필수 컬럼:

```text
학급, 학번, 이름
```

처리 규칙:

- 학급명이 없으면 오류 처리한다.
- 학번이 없으면 오류 처리한다.
- 이름이 없으면 오류 처리한다.
- 학번은 교사 계정 내에서 고유해야 한다.
- 파일에 있는 학급이 기존에 없으면 자동 생성한다.
- 기존 학번이 다시 업로드되면 업데이트 또는 중복 오류 중 하나를 선택해야 한다.
- 초기 정책은 중복 오류로 처리한다.
- 업로드 완료 후 학생별 임시 비밀번호를 자동 생성한다.

업로드 결과 화면:

- 총 행 수
- 성공 등록 수
- 오류 행 수
- 오류 사유
- 발급된 비밀번호 다운로드

### 4.3 모둠 생성

교사는 학급별 기본 모둠을 만들 수 있다.

생성 방식:

- 직접 생성
- 무작위 자동 생성

자동 생성 옵션:

- 모둠 수 기준
- 모둠당 인원수 기준

자동 생성 후 교사는 학생을 다른 모둠으로 이동할 수 있다.

활동별 모둠:

- 활동 생성 시 학급 기본 모둠을 복사해서 사용할 수 있다.
- 활동 안에서 수정된 모둠은 학급 기본 모둠에 영향을 주지 않는다.

### 4.4 See-Think-Wonder 활동 생성

교사는 다음 정보를 입력한다.

- 활동 제목
- 운영 위치
  - 학급 우선: 학급, 과목
  - 과목 우선: 과목, 대상 학급
- 안내 텍스트
- 중심 자료 1개
  - 이미지
  - PDF
- 활동별 모둠 사용 여부
- 공개 상태
- 생성 버튼

생성 후 시스템은 다음을 만든다.

- 활동 ID
- 짧은 활동 코드
- QR 코드
- 학생 접속 URL

### 4.5 학생 활동 바로 접속

```text
QR 스캔 또는 활동 코드 입력
→ 활동 인증 화면
→ 학번 + 이름 + 비밀번호 입력
→ 활동 화면 진입
```

인증 성공 조건:

- 활동이 존재한다.
- 학생이 해당 활동의 대상 학급에 속한다.
- 학번, 이름, 비밀번호가 일치한다.

### 4.6 학생 내 활동 목록 접속

```text
학생 접속 페이지
→ 학번 + 이름 + 비밀번호 입력
→ 내 활동 목록
→ 활동 선택
```

목록 표시:

- 진행 중 활동
- 마감된 활동
- 활동 제목
- 과목/학급
- 상태
- 제출 상태
- 모둠명

### 4.7 학생 See-Think-Wonder 작성

학생은 중심 자료를 보고 세 영역에 카드를 추가한다.

- See
- Think
- Wonder

저장 정책:

- 카드 추가/수정/삭제 시 자동 저장한다.
- 학생은 제출 버튼을 눌러 완료 상태를 표시한다.
- 제출 후에도 활동 마감 전까지 수정할 수 있다.
- 제출 후 수정하면 제출 상태는 유지하고 제출 후 수정됨 표시를 남긴다.
- 마감 후에는 읽기 전용이다.

### 4.8 교사 결과 조회

학급 전체 보기:

- See / Think / Wonder 컬럼별 카드 목록
- 학생 이름 표시
- 익명 보기 토글
- 태그 필터
- 제출 상태 필터
- 학급 필터
- 학생 클릭 시 개인 결과 화면 이동

학생별 보기:

- 학생 정보
- 제출 상태
- 제출 후 수정됨 여부
- See / Think / Wonder 카드
- 이전 학생/다음 학생 이동
- 개인 AI 분석 버튼
- AI 분석 공개 버튼

## 5. 기능 요구사항

### 5.1 인증

교사 인증:

- 이메일/비밀번호 기반 로그인을 지원한다.
- 교사 세션은 서버에서 검증되어야 한다.
- 교사 데이터는 교사별로 분리되어야 한다.

학생 인증:

- 학생은 일반 Auth 사용자로 만들지 않는다.
- 학생 인증은 서버 API에서 학번, 이름, 비밀번호를 검증한다.
- 학생 비밀번호는 평문 저장하지 않는다.
- 학생 세션은 짧은 수명의 전용 세션 토큰으로 관리한다.

### 5.2 파일 업로드

지원 파일:

- 엑셀
- CSV
- 이미지
- PDF

제한:

- 파일 크기 제한을 둔다.
- 허용 MIME 타입을 검증한다.
- 업로드 파일은 교사별 접근 권한을 적용한다.
- 학생 활동 자료 파일은 대상 활동 참여자와 교사만 접근할 수 있어야 한다.

### 5.3 활동 코드와 QR

- 활동 코드는 짧고 읽기 쉬워야 한다.
- 활동 코드는 교사 계정 전체 또는 전역에서 충돌하지 않아야 한다.
- QR 코드는 활동 접속 URL을 포함한다.
- 활동 코드가 노출되어도 학생 인증 없이는 활동에 접근할 수 없어야 한다.

### 5.4 자동 저장

- 카드 작성은 자동 저장한다.
- 저장 중, 저장 완료, 저장 실패 상태를 표시한다.
- 네트워크 실패 시 사용자가 내용을 잃지 않도록 임시 상태를 유지한다.
- 저장 실패 시 재시도할 수 있어야 한다.

### 5.5 태그

기본 태그:

- 좋은 관찰
- 흥미로운 질문
- 오개념
- 토론거리

요구사항:

- 교사는 사용자 정의 태그를 만들 수 있다.
- 카드는 여러 태그를 가질 수 있다.
- 태그별 필터링을 지원한다.
- 태그는 기본적으로 교사에게만 보인다.
- 교사가 공개한 태그만 학생에게 보인다.

### 5.6 AI 분석

개인 분석:

- 제출 완료 학생에게만 실행할 수 있다.
- 분석 결과는 저장한다.
- 제출물이 바뀌면 갱신 필요 표시를 한다.
- 학생용 피드백과 교사용 해석을 분리한다.
- 교사가 승인한 학생용 피드백만 학생에게 공개한다.

학급 분석:

- 교사가 언제든 실행할 수 있다.
- 제출률을 보고서에 포함한다.
- 제출률이 낮으면 일부 학생 결과 기준임을 표시한다.
- 보고서는 저장한다.
- 제출물이 바뀌면 갱신 필요 표시를 한다.

See-Think-Wonder 학급 분석 항목:

- 많이 나온 관찰
- 흥미로운 질문
- 오개념 가능성
- 더 탐구할 주제
- 다음 발문 제안
- 추천 사고 루틴
- 제출률

## 6. 화면 요구사항

### 6.1 교사 대시보드

목적:

- 최근 활동과 해야 할 일을 빠르게 확인한다.

주요 요소:

- 최근 활동 목록
- 진행 중 활동 수
- 마감된 활동 수
- 최근 제출 현황
- 학급 바로가기
- 활동 만들기 버튼

### 6.2 학생 일괄 등록 화면

주요 요소:

- 업로드 영역
- 지원 양식 안내
- 샘플 CSV 다운로드
- 업로드 미리보기
- 오류 행 표시
- 등록 결과 요약
- 비밀번호 다운로드

### 6.3 학급/모둠 관리 화면

주요 요소:

- 학급 목록
- 학생 목록
- 학생 검색
- 비밀번호 재발급
- 기본 모둠 보드
- 자동 모둠 생성 버튼
- 모둠 수/모둠당 인원수 선택
- 학생 드래그 이동 또는 선택 이동

### 6.4 활동 목록 화면

주요 요소:

- 전체/진행 중/마감됨 필터
- 상태 뱃지
- 과목/학급 필터
- 활동 코드
- QR 보기
- 결과 보기
- 마감/다시 열기

### 6.5 See-Think-Wonder 활동 생성 화면

주요 요소:

- 제목 입력

### 7.2 엔티티 설명

teachers:

- 교사 로그인 계정이다.
- 인증 시스템의 사용자와 연결된다.

teacher_profiles:

- 교사 운영 방식과 기본 설정을 저장한다.
- 운영 방식 값은 class_first 또는 subject_first이다.

classes:

- 교사가 관리하는 학급이다.

students:

- 교사가 등록한 학생이다.
- 학번, 이름, 비밀번호 해시를 가진다.
- 학번은 교사 계정 내에서 고유하다.

subjects:

- 교사가 관리하는 과목 또는 폴더이다.

groups:

- 학급 기본 모둠이다.

group_members:

- 학급 기본 모둠의 학생 배정이다.

activities:

- 교사가 만든 활동이다.
- 루틴 종류, 제목, 안내 텍스트, 상태, 활동 코드를 가진다.

activity_class_assignments:

- 하나의 활동이 여러 학급에 배정될 수 있으므로 활동과 학급을 연결한다.

activity_groups:

- 활동별 모둠이다.
- 기본 모둠을 복사해서 생성할 수 있다.

activity_group_members:

- 활동별 모둠의 학생 배정이다.

routine_templates:

- See-Think-Wonder 같은 루틴 정의를 저장한다.

submissions:

- 학생별 활동 제출 상태이다.
- 작성 중, 제출 완료, 제출 후 수정됨 상태를 가진다.

routine_cards:

- See / Think / Wonder 카드이다.
- 카드 타입, 내용, 작성자, 순서를 가진다.

tags:

- 기본 태그와 교사 사용자 정의 태그이다.

card_tags:

- 카드와 태그의 연결이다.
- 공개 여부를 가진다.

ai_reports:

- 개인 또는 학급 AI 분석 결과이다.
- 보고서 종류, JSON 결과, 공개 여부, 갱신 필요 여부를 가진다.

files:

- 이미지, PDF, 업로드 파일 메타데이터이다.

## 9. 비기능 요구사항

### 9.1 보안

- 교사별 데이터 접근을 분리한다.
- 학생 비밀번호는 해시로 저장한다.
- 학생 세션 토큰은 짧은 수명으로 발급한다.
- 파일 접근은 권한 검사를 거친다.
- 공개 활동 코드만으로 학생 데이터가 노출되면 안 된다.
- AI 분석 요청에는 필요한 학생 산출물만 포함한다.

### 9.2 개인정보

- 학생 데이터는 최소 항목만 저장한다.
- 초기 필수 학생 정보는 학급, 학번, 이름, 비밀번호 해시이다.
- AI 분석 결과 공개는 교사 승인 후에만 가능하다.
- 학생에게 교사용 해석이 노출되지 않아야 한다.

### 9.3 성능

- 학급 전체 결과 화면은 일반적인 한 학급 규모인 20~35명 기준으로 빠르게 렌더링되어야 한다.
- 카드 수가 많아질 경우 페이지네이션 또는 가상 스크롤을 고려한다.
- AI 분석은 비동기 작업으로 처리하고 생성 중 상태를 표시한다.

### 9.4 접근성

- 모바일, 태블릿, 데스크톱에서 사용 가능해야 한다.
- QR 접속 후 모바일 작성 흐름이 자연스러워야 한다.
- 버튼, 입력창, 탭, 카드에는 명확한 포커스 상태가 있어야 한다.
- 색상만으로 상태를 구분하지 않는다.

## 10. 추천 기술 스택

### 10.1 최종 추천

초기 제품에는 다음 스택을 추천한다.

```text
Frontend / Full-stack: Next.js App Router + TypeScript
UI: Tailwind CSS + shadcn/ui + lucide-react
Database: Firebase SQL Connect / Firebase Data Connect backed by Cloud SQL for PostgreSQL
Auth: Firebase Authentication for teachers
Student auth: custom server-side credential verification
Storage: Cloud Storage for Firebase
Server logic: Next.js Route Handlers / Server Actions
AI: Gemini API via Google GenAI SDK, with Vercel AI SDK as optional wrapper
Validation: Zod
CSV/Excel: SheetJS/xlsx 또는 exceljs
QR: qrcode
Deployment: Vercel
```

### 10.2 선택 이유

Next.js App Router:

- 풀스택 웹앱을 하나의 프로젝트에서 구성하기 좋다.
- 교사용 서버 렌더링 화면과 학생용 인터랙티브 화면을 함께 만들 수 있다.
- App Router는 파일 기반 라우팅, Server Components, Server Functions 등을 지원한다.

Firebase SQL Connect / Firebase Data Connect:

- Cloud SQL for PostgreSQL 기반의 관계형 데이터베이스를 Firebase 생태계에서 사용할 수 있다.
- 학급, 학생, 활동, 제출물처럼 관계형 데이터가 많은 제품에 적합하다.
- 스키마, 쿼리, 뮤테이션을 명시적으로 정의하고 타입 안전한 SDK를 생성할 수 있다.
- Firebase Authentication과 연결해 교사 인증 기반 접근 제어를 구성할 수 있다.
- Firebase 생태계 안에서 Auth, Storage, AI 연동을 일관되게 가져갈 수 있다.

Gemini API:

- 개인/학급 분석 보고서는 구조화된 JSON 결과가 필요하다.
- Gemini API는 JSON Schema 기반 구조화 출력을 지원한다.
- Google GenAI SDK는 JavaScript/TypeScript에서 Gemini 모델 호출에 적합하다.
- Vercel AI SDK는 여러 모델 제공자를 추상화해야 할 때 선택적으로 사용할 수 있다.

Vercel:

- Next.js 배포와 운영이 단순하다.
- Route Handler 기반 API와 서버 작업을 운영하기 쉽다.

### 10.3 기술적 주의사항

학생 인증:

- 학생을 Firebase Authentication 사용자로 만들지 않는다.
- 학생 인증은 별도 테이블과 서버 API로 처리한다.
- 학생 비밀번호는 bcrypt 또는 argon2로 해시한다.
- 학생 세션은 HTTP-only 쿠키로 관리한다.

Firebase SQL Connect / Data Connect:

- Firebase 문서에서는 관계형 DB 제품이 Data Connect로도 표기된다. 최근 방향은 SQL Connect 명칭으로 전환되는 흐름이 있으므로 구현 시 현재 공식 문서를 다시 확인한다.
- 교사용 데이터 접근은 Firebase Authentication 사용자와 연결된 쿼리/뮤테이션 권한으로 제한한다.
- 학생 인증이 커스텀이므로 학생 접근은 Next.js 서버 API를 거쳐 권한을 확인하는 방식이 안전하다.
- Data Connect/SQL Connect에서 직접 지원하지 않는 관리자성 작업은 서버 환경의 Firebase Admin SDK 또는 Google Cloud 권한으로 처리한다.

AI 분석:

- 분석 결과는 JSON 스키마로 저장한다.
- 보고서 생성 입력에는 학생 이름을 최소화하거나 내부 식별자로 대체할 수 있다.
- AI 결과는 교사 검토 후 공개한다.
- 제출물 수정 시 기존 보고서에 stale 상태를 표시한다.
- Gemini 구조화 출력이 실패하거나 스키마 검증을 통과하지 못하면 재시도 또는 실패 상태로 저장한다.

파일:

- 이미지/PDF 업로드는 Cloud Storage for Firebase에 저장한다.
- 파일 URL은 공개 버킷보다 권한 확인 또는 제한된 다운로드 URL을 사용하는 방식이 적합하다.
- 업로드 MIME 타입과 크기 제한을 적용한다.
